# Get quarterly agricultural production index

Fetches quarterly production index data across five endpoints.

**Primary search modes (choose one):**

- `sector = TRUE` — list all sectors (no other parameters)

- `category_code` — search by product category

- `group_code` — search by product group

- `product_code` — search by product name

- `year_th` + `quarter` — fetch all commodities for a specific quarter
  (standalone)

**Optional filter (combine with category_code, group_code, or
product_code):**

- `quarter` — narrow results to a specific quarter (1-4)

## Usage

``` r
get_production_index_quarter(
  category_code = NULL,
  group_code = NULL,
  product_code = NULL,
  year_th = NULL,
  quarter = NULL,
  sector = FALSE,
  api_key = NULL
)
```

## Arguments

- category_code:

  Category code (see
  [`show_index_categories()`](https://kanthjs.github.io/talatThaiR/reference/show_index_categories.md),
  e.g. "LIVESTOCK")

- group_code:

  Group code (see
  [`show_index_groups()`](https://kanthjs.github.io/talatThaiR/reference/show_index_groups.md),
  e.g. "OIL_CROP")

- product_code:

  Product code (see
  [`show_quarter_products()`](https://kanthjs.github.io/talatThaiR/reference/show_quarter_products.md),
  e.g. "GARLIC_DRY_MIX"). Note: quarterly prices use a separate product
  map from monthly prices.

- year_th:

  Thai Buddhist year (e.g. 2568). Used with `quarter` for the `/all`
  endpoint.

- quarter:

  Quarter as integer 1-4 (e.g. 1 or 4). Optional filter for
  category/group/product; required together with `year_th` for
  standalone `/all` mode.

- sector:

  Logical. If `TRUE`, fetches the full sector reference list. No other
  parameters should be specified. Default: `FALSE`.

- api_key:

  API key (if required)

## Value

A data.frame of quarterly production index records
